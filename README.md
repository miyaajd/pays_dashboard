# pays_dashboard

<프로젝트 소개>

* 결제/가맹점 데이터를 기반으로 매출 분석 및 조회 기능을 제공하는 관리자 대시보드

* 올페이즈 과제전형 제출용

* 웹 화면에 최적화 되었습니다 


<기술 스택>

* Next.js (App Router)

* TypeScript

* Tailwind CSS

* Material UI (MUI)

* Chart.js

* API 연동


<기능 목록>

1) 대시보드

* 총 매출액 / 총 거래건수 / 가맹점 수 표시

* 결제 성공·실패·환불 수량 표시

* 결제 성공률 차트 시각화

* 최근 거래내역 (스크롤 리스트)

* 가맹점 매출순 Ranking (높은순/낮은순 정렬)

2) 거래내역 조회 페이지

* 상태별 필터 (전체/성공/실패/환불)

* 가맹점별 필터

* 날짜별 조회

* 검색(결제코드/가맹점명)

* 페이지네이션

* URL 쿼리로 상태분기 (예: ?status=SUCCESS)


<직접 디자인한 이유 & UI/UX 포인트>

// 디자인 의도 //

* 금융 데이터(결제, 매출, 가맹점 정보)를 다루는 관리자 화면이기 때문에 직관적이고 빠르게 정보가 읽히는 구조를 최우선으로 설계함.

* 카드형 통계, 테이블형 리스트, 차트 등 데이터 가독성 중심의 UI 구성.

* 복잡한 관리자 화면이 답답해 보이지 않도록 여백을 넉넉하게 배치하고, 컬러 포인트는 최소화하여 집중도를 높임.

// 주요 UI/UX 포인트 //

* 결제 상태별 컬러 일관성 유지

* 성공(파랑), 실패(핑크), 환불(그레이)을 전체 화면에서 동일 컬러 로직으로 표시해 데이터 해석이 빠르고 명확하도록 구성.

* 카드형 통계 + 테이블 구조의 하이브리드 디자인

* 대시보드의 핵심 데이터(매출, 거래건수, 가맹점수)를 카드로 강조하고 상세조회는 테이블에서 이루어지도록 정보 레벨을 단계적으로 설계.

* 필터 + 테이블 + 페이지네이션의 논리적 흐름

* 검색/필터링 → 결과 리스트 → 페이지 이동 이 세 단계가 자연스럽게 연결되도록 UI를 구성하여 UX 피로도를 줄임.

* 차트 단순화 및 색상 최적화

* Pie Chart를 사용해 성공/실패/환불 비율을 직관적으로 파악하도록 구성.

* 대시보드의 총 매출액, 총 거래건수 클릭시 → 거래내역 조회 페이지 이동

* 대시보드의 결제성공, 실패, 환불 클릭시 → 각 항목별 필터링 된 거래내역 페이지로 이동
